lab 35 Merging Back to Master

Goals

  • We’ve kept our greet branch up to date with master (via rebase), now let’s merge the greet changes back into the master branch.

Merge greet into master 01

Execute:

git checkout master
git merge greet

Output:

$ git checkout master
Switched to branch 'master'
$
$ git merge greet
Updating e7f49d0..f30c158
Fast-forward
 Rakefile       |    2 +-
 lib/greeter.rb |    8 ++++++++
 lib/hello.rb   |    6 ++++--
 3 files changed, 13 insertions(+), 3 deletions(-)
 create mode 100644 lib/greeter.rb

Because the head of master is a direct ancestor of the head of the greet branch, git is able to do a fast-forward merge. When fast-forwarding, the branch pointer is simply moved forward to point to the same commit as the greeter branch.

There will never be conflicts in a fast-forward merge.

Review the logs 02

Execute:

git hist

Output:

$ git hist
* f30c158 2012-04-28 | Updated Rakefile (HEAD, master, greet) [Jim Weirich]
* f198b1f 2012-04-28 | Hello uses Greeter [Jim Weirich]
* cc84df4 2012-04-28 | Added greeter class [Jim Weirich]
* e7f49d0 2012-04-28 | Added README [Jim Weirich]
* cb1bb8e 2012-04-28 | Added a Rakefile. [Jim Weirich]
* f1d027c 2012-04-28 | Moved hello.rb to lib [Jim Weirich]
* d87139a 2012-04-28 | Add an author/email comment [Jim Weirich]
* 3d2a00b 2012-04-28 | Added a comment (v1) [Jim Weirich]

The greet and master branches are now identical.
